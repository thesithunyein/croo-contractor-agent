Orchestration Flow

From goal to composed result — every step is on-chain, verified, and tamper-evident.

1

Plan

Goal → step DAG (keyword-routed, LLM-pluggable)

2

Route

Resolve each capability to cheapest specialist serviceId

3

Hire

negotiateOrder → payOrder → getDelivery over WebSocket

4

Verify

Check deliverable (non-empty, schema, required fields)

5

Compose

Merge verified outputs + proof bundle → deliver to caller

Try it
npm run demo -- "Diagnose Solana error 0x1770 and summarize the fix"

What Makes It Different

The Contractor exercises the full CAP surface — simultaneously a provider and requester.

Dual-Role Architecture

Provider + Requester in one agent — the Contractor is both hired and hires

Verification-First

Every sub-delivery is checked (text validity, JSON schema, required fields) before composition

Tamper-Evident Proof Bundle

Each composed result includes { orderId, resultHash, txHash } per sub-order

Spend Guardrails

MAX_USDC_PER_ORDER / MAX_USDC_PER_JOB caps prevent overspend

Fan-Out Mode

Hire ALL registered agents in one run — built for racking up unique counterparties

Partner Integration

Add other teams' agents via partners.json — zero code changes needed

A2A Composability

The Contractor doesn't just hire its own specialists — it hires other teams' agents too. Zero code changes.
